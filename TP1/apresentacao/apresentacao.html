1.

Introdução

O objetivo central foi projetar e desenvolver uma biblioteca de grafos em C++ que fosse modular, reutilizável e eficiente, com foco em grafos não direcionados. A biblioteca deveria não apenas representar grafos, mas também implementar um conjunto robusto de algoritmos de análise.

Principais Funcionalidades Implementadas:

Múltiplas Representações Internas

Quatro formas de armazenar o grafo, escolhendo a mais adequada ao caso.

  • Matriz de Adjacência (Clássica)

    std::vector<std::vector<bool>>. Didática, porém O(V²) em memória.

  • Matriz de Adjacência (Triangular)

    Vetor 1D com metade da matriz, economizando ~50% de memória.

  • Lista de Adjacência (Simples)

    std::vector<std::list<int>>. Intuitiva e O(V+E) em memória.

  • Lista de Adjacência (CSR)

    Construção em duas fases; estrutura compacta e de alto desempenho.

Algoritmos de Busca

Implementações fundamentais para exploração do grafo.

  • Busca em Largura (BFS)

    Fila (queue) para explorar o grafo camada por camada.

  • Busca em Profundidade (DFS)

    Pilha (stack) para exploração profunda de um ramo e backtracking.

Análise de Conectividade

Medições estruturais do grafo baseadas em buscas.

  • Distâncias e Componentes Conexas

    Cálculo via BFS para identificar alcances e agrupamentos.

  • Diâmetro

    Cálculo exato (força bruta) e versão aproximada para grafos grandes.

Carregamento e Relatórios

Entrada de dados e registro dos resultados.

  • Importação

    Leitura de grafos a partir de arquivos .txt.

  • Relatórios e Estudos de Caso

    Geração de métricas e menus iterativos para experimentos.

2.

Arquitetura e Design

A biblioteca foi construída sobre princípios de orientação a objetos para garantir flexibilidade e manutenibilidade, separando claramente as responsabilidades.

Estrutura de Classes

  • Grafo: A interface pública (fachada) que oferece uma API intuitiva.
  • Representação: Interface base com 4 implementações concretas (Matrizes e Listas).
  • Algoritmo: Classes especialistas (BFS, DFS) que contêm a lógica e são delegadas pela classe Grafo.

Organização de Pastas

biblioteca/
├── /interface/       # API Pública (Grafo.h)
├── /representacao/   # Estruturas de Dados
└── /algoritmos/      # Lógica dos Algoritmos

estudos/
└── /benchmark/       # Programas de teste

5.

Conclusão e Justificativa de Resultados

Os estudos de caso demonstraram de forma conclusiva os trade-offs entre as diferentes estruturas de dados. A análise de complexidade teórica, apresentada na tabela abaixo, foi validada pelos resultados práticos de memória e tempo.

Análise de Complexidade Teórica

Operação Matriz de Adjacência Lista de Adjacência
Espaço (Memória) O(V²) O(V+E)
Verificar Aresta (u,v) O(1) O(grau(V))
Listar Vizinhos(u) O(V) O(grau(V))

Justificativa da Performance Superior da Lista Otimizada (CSR)

Embora tanto a Lista Simples quanto a CSR tenham a mesma complexidade teórica (O(V+E)), os benchmarks mostraram que a implementação CSR foi consistentemente mais rápida, chegando a ser mais de 20 vezes superior. Isso se deve a dois fatores principais de baixo nível:

Localidade de Cache

A CSR armazena todos os vizinhos de um grafo em um único vetor contíguo na memória. Quando o algoritmo (BFS/DFS) precisa listar os vizinhos de um vértice, ele acessa um bloco de memória sequencial. Isso é extremamente eficiente para a CPU, pois maximiza o uso da memória cache e minimiza o tempo de espera. Em contraste, a std::vector<std::list<int>> da Lista Simples armazena os vizinhos em nós de lista ligada que podem estar espalhados por toda a RAM, causando "cache misses" frequentes e tornando o acesso mais lento.

Menor Overhead de Memória

Cada elemento de uma std::list carrega um custo extra de memória para ponteiros (para o nó anterior e o próximo). A CSR elimina esse overhead ao usar um único vetor, tornando-a não apenas mais compacta (como visto no Estudo de Caso 1), mas também mais rápida de percorrer.

Conclui-se que para grafos esparsos, a representação de Lista de Adjacência é a escolha correta, e a implementação com a técnica CSR é superior em todos os cenários práticos devido à sua excelente otimização de acesso à memória.
